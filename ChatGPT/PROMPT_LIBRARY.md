# Prompt Library

Store reusable prompts/patterns you can copy/paste.

## Meta
- **Goal:** Make prompts consistent, testable, and easy to reuse.
- Prefer: clear inputs, constraints, outputs, and acceptance criteria.

## Templates

### 1) Spec → Plan → Deliverable
**Context:** <what we’re building>
**Constraints:** <OS, versions, time, limits>
**Deliverable:** <files, formats, commands>
**Acceptance:** <how we verify>

### 2) Diagnose / Debug
**Symptom:** <what fails>
**Environment:** <OS, versions>
**Logs:** <paste>
**Expected:** <what should happen>
**Ask:** root cause + minimal fix + verification steps

### 3) Research / Compare
**Question:** <what>
**Scope:** <in / out>
**Decision factors:** <cost, perf, risk>
**Output:** recommendation + tradeoffs + citations
